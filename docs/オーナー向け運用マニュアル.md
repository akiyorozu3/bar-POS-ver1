# バーPOS オーナー向け 運用マニュアル

お店のオーナー様向けの運用説明書です。ログイン情報、日々の使い方、メニュー・キャスト・料率の設定、売上の見方、打刻（勤怠）、レジ締めまでをまとめています。

---

## 0. ログイン情報（最初に控えてください）

> ⚠️ この欄は納品時に実際の値を記入してください。パスワードは外部に漏れないよう管理してください。

| 項目 | 内容 |
|---|---|
| アプリのURL | `https://bar-pos-b1993.web.app` |
| オーナー用 ログインID | `owner` |
| オーナー用 パスワード | `password` |
| スタッフ用 ログインID | `staff` |
| スタッフ用 パスワード | `000000` |

> ⚠️ **セキュリティ注意**: 上記パスワードは推測されやすいため、できるだけ早めに変更を推奨します（特にオーナー）。
> 公開URLは誰でもアクセスできるため、オーナーのパスワードが破られると売上・バック額がすべて閲覧されます。
> 変更は Firebase Console → Authentication → 対象ユーザー → パスワード再設定 から行えます（変更後はこの表も更新してください）。

- iPad のホーム画面に追加すると、アプリのように使えます（Safari でURLを開く → 共有 → 「ホーム画面に追加」）。
- **オーナーとスタッフでアカウントが分かれています。** スタッフ用では売上やバックは見られません。
- オーナー用パスワードは、スタッフに教えないでください（売上が見られてしまいます）。

## 2. 画面の構成

ログイン後、画面上部のタブで切り替えます（オーナーは6つ、スタッフは2つ）。

- **注文入力** … 席ごとに注文を取る画面
- **会計** … 支払いを確定する画面。いつでも開けて、上部の「未会計の卓」チップで卓を切り替えられます
- **売上管理**（オーナー）… 売上集計・キャストバック・CSV出力・手数料/バック率/消費税の設定・取引明細・日払い/大入・レジ締め
- **メニュー管理**（オーナー）… ドリンク等メニューの管理
- **キャスト管理**（オーナー）… キャスト（ニックネーム・本名）の管理
- **打刻管理**（オーナー）… 出退勤の集計・修正・勤務CSV

画面いちばん上の黒い帯には次があります。

- **日付**（記録される営業日）… 通常は今日。過去日に変えると帯がオレンジになり「遡及入力」モードに（「今日に戻す」で戻る／リロードでも今日に戻ります）。
- **打刻** ボタン … キャストの出勤・退勤を記録（オーナー・スタッフ共通）。詳細は「9. 打刻」。
- **ログアウト**

## 3. 初期設定（導入時に一度だけ）

オーナーでログインして行います。

### 3-1. メニューを登録する
1. 上部「**メニュー管理**」を開く
2. まだ何も無ければ「**デフォルトメニューを投入**」を押すと初期メニューが入ります
3. あとは画面から自由に追加・編集・削除（下記4参照）

### 3-2. キャストを登録する
1. 上部「**キャスト管理**」を開く
2. 「**初期キャストを投入**」を押すか、**ニックネーム**（必要なら**本名**も）を入れて「＋追加」

### 3-3. 消費税・手数料・バック率を設定する
1. 「**売上管理**」→「**手数料/バック**」ボタン
2. 以下をまとめて設定して「保存」
   - **価格・消費税の扱い**:「税込で登録・加算しない」/「税抜で登録・会計時に加算」
   - **消費税率**（%）
   - **カード手数料 / QR手数料**（%）
   - **卓バック率**（%）… 会計の合計にかけて卓の担当に入る
   - **キャストドリンクバック率**（%）… キャストドリンクの料金にかけてその担当に入る

> 詳しくは下の「6. 売上の見方」内の **消費税の設定** と **バックの考え方** を参照。

---

## 4. メニューの管理（オーナー）

「メニュー管理」画面で行います。

- **追加**: 上部でカテゴリを選び、メニュー名・金額を入れて「＋追加」。「本日限定」を選ぶと当日限りのメニューになります。
- **編集**: 一覧の各行で名前・金額を直接書き換え、「保存」を押す。
- **削除**: 各行の「削除」。
- カテゴリ: **セット / ショット / シャンパン / キャストドリンク / ウイスキー / カクテル / ビール / フード**。
- 金額の入力単位は消費税の設定で変わります（「税込で登録」モードなら**税込**、「税抜で登録」モードなら**税抜**。入力欄の表示も自動で切り替わります）。

> 金額が未入力だと「＋追加」は押せません（金額欄は青枠で目立たせています）。
> 本日限定メニューは、注文画面の「本日メニュー編集」からも追加・削除できます。
> 「キャストドリンク」は専用カテゴリです。このカテゴリの商品にだけ「ドリンクバック」が付きます（「6.」内のバックの考え方を参照）。

---

## 5. キャストの管理（オーナー）

「キャスト管理」画面で行います。

- **追加**: **ニックネーム**を入れて「＋追加」。必要なら**本名**も入力（Enter でも追加可）。
- **変更**: 各行のニックネーム・本名を書き換えて「保存」。
- **削除**: 各行の「削除」。

ニックネームと本名の使い分け:
- **ニックネーム（源氏名）**… 店内表示（卓の担当・ドリンク担当・打刻の選択肢）に使います。
- **本名**… 給与計算・CSV に使います（本名が未入力ならニックネームが使われます）。

注意:
- ここで登録したキャストが、注文画面の「卓の担当」やキャストドリンクの「ドリンク担当」、打刻のプルダウンの選択肢になります。
- **キャストを削除・改名しても、過去の売上・バック・打刻の記録は変わりません**（記録時点の名前がそのまま残ります）。改名は以降の新しい記録から反映されます。

## 6. 売上の見方（オーナー）

「売上管理」画面で確認します。上部で **今日 / 今週 / 今月** を切替（期間は画面上部の日付が基準。「今週」は直近7日間）。

- **売上合計**（税込）/ 件数 / 客単価
- **決済手数料**（カード・QRの手数料合計）
- **実際の入金合計**（売上 − 手数料。実際に手元に入る金額）
- **支払い方法別 実入金**（現金・カード・QRごとの件数・売上・手数料・実入金）
- **支払い方法の割合**
- **キャストバック集計**（キャストごとの卓数・売上・バック額）

### 消費税の設定
「手数料/バック」パネルで、消費税の扱いと税率を設定できます。
- **税込で登録・加算しない**: メニューに登録した金額がそのままお客様の支払額。消費税は上乗せ・表示しません。
- **税抜で登録・会計時に加算**: 登録は税抜。会計時に消費税率（%）を上乗せして合計を出します。
- **消費税率**（%）も設定可（既定10%、0%も可）。
- 切替に連動して、メニュー登録欄やレシートの「税込/税抜」表記も自動で変わります。

### バックの考え方（2種類）
バックは次の2つが**別々に**計算され、合算されます。

1. **卓バック**：会計の**合計 ×「卓バック率」** → その卓の担当キャストに入る。
   - 卓の担当が**複数いる場合は頭割り**（例: 合計3,200×10%=320を2名で 160ずつ）。
2. **キャストドリンクバック**：キャストドリンクの**料金 ×「ドリンクバック率」** → その**ドリンクの担当**に入る（卓バックとは別に上乗せ）。

ポイント:
- キャストドリンクの料金は**卓バックの合計にも含まれます**。たか子の卓にゆう子のキャストドリンクがあれば、**たか子には卓バック（ドリンク料金込み）、ゆう子にはドリンクバック**が両方付きます。
- 卓の担当が未設定の卓は卓バックが付きません（集計には表示）。

## 7. 会計済み伝票の閲覧・編集・削除（オーナー）

「売上管理」→「**取引明細**」で、選択中の期間の会計済み伝票を一覧できます（**このボタンはオーナーだけに表示**。会計済みデータはオーナーのみ閲覧できます）。

- **閲覧**: 各行の「閲覧」で内容（明細・金額・支払方法・担当）を確認。
- **編集**: 「編集」を押すと、その伝票を**注文入力の卓として復活**させ、作り直せます。作り直して再度会計してください。
- **削除**: 「削除」で取引を取り消せます。

> **レジ締め済みの日の伝票は、編集・削除できません**（先に「締め解除」が必要）。誤操作防止のためです。

---

## 8. 日払い・大入（オーナー）

その日に現金で渡した**日払い**や**大入り**を記録します。「売上管理」→「**日払い/大入**」。

- キャストと種別（日払い / 大入）、金額を入れて追加。
- その日の一覧が出て、削除もできます。
- ここで記録した額は **レジ締めの「金庫に残る現金」計算に反映**されます（下記9）。

---

## 9. 打刻（出退勤の記録）

画面右上の「**打刻**」ボタンから、キャストの出勤・退勤を記録します（**オーナー・スタッフ共通で押せます**）。

1. 「打刻」を開く
2. **キャストをプルダウンで選ぶ**（キャスト管理の登録が選択肢になります）
3. 「**出勤**」または「**退勤**」を押す → その瞬間の時刻で記録
4. 下に**今日の打刻一覧**が出ます

> 打刻は「出勤」「退勤」の**イベント記録**です。勤務時間の集計や修正は次の「打刻管理」で行います（打刻管理はオーナーのみ）。

---

## 10. 打刻管理（オーナー）

「**打刻管理**」タブで、勤怠の集計・確認・修正を行います。

- **期間で検索**して、**勤務一覧**（出勤〜退勤のペア＝1勤務）を表示。日付をまたぐ深夜勤務も、出勤した日の勤務として集計されます。
- **要修正サマリー**: 退勤し忘れ（出勤だけ残っている）や、出勤が無い退勤などの**不整合を、どのキャストで起きているか**一目で分かるように表示します。
- **修正**: 打刻の時刻を直したり、抜けた打刻を**手動で追加**できます（要修正のみに絞り込み可）。
- **勤務CSV**: 期間の勤務（キャスト・出勤・退勤・勤務時間）を出力。給与計算に使えます（本名で出力）。

> 出退勤の**修正・追加・削除はオーナーのみ**です。スタッフは打刻（出勤/退勤を押す）だけできます。

---

## 11. CSV 出力（オーナー）

「売上管理」画面から、選択中の期間のデータを出力できます。

- **売上CSV** … 取引明細（日時・席・支払方法・売上・手数料・実入金・主担当など）
- **バックCSV** … キャスト別の担当件数・売上(税抜)・バック額

勤務（打刻）の CSV は「打刻管理」から出力します（上記10）。
いずれも Excel やスプレッドシートでそのまま開けます（文字化けしません）。

## 12. レジ締め（オーナー）

営業終了時に、その日の売上を確定します。「**売上管理**」→「**レジ締め**」ボタンで開きます。

表示される内容:
- 本日の **現金 / カード / QR 売上・売上合計・決済手数料・実際の入金合計・バック合計・日払い・大入**
- **金庫に残る現金**（現金売上 − 現金で払ったバック・日払い・大入 の目安）

操作:
- **未会計の卓がある場合は警告**が出ます。先に会計を済ませてから締めてください。
- **「本日を締める」** を押すと確定し、**その日の会計入力ができなくなります**（スタッフが会計しようとしても「締め済み」でブロック）。
- 間違えた場合は **「締め解除」** で元に戻せます（オーナーのみ）。

> 「その日」は画面上部の日付（営業日）ごとです。遡及入力で過去日を締めることもできます。締め済みの日に会計・取引編集をしようとするとブロックされます。

## 13. 注文まわりの便利機能（参考）

スタッフ向けの操作ですが、オーナーも把握しておくと安心です。

- **卓の開き方**: 「**＋追加**」で新規、または上部の**定型ボタン**（C1〜C8 などよく使う卓）をワンタップで開けます。定型は「**定型編集**」（オーナー）で登録・変更できます。
- **卓を閉じる**: 各卓の「**✕**」で閉じられます。**会計を完了した卓は自動で消えます**。
- **割引**: 「フリー入力」タブに**割引**の行があり、伝票に割引を入れられます。
- **現金受取額**: 会計の現金支払いで、**受け取った金額を自由に入力**でき、お釣りが自動計算されます。

## 14. スタッフを増やしたいとき

スタッフ用アカウントは「`staff`」1つを共有する想定です。別アカウントを増やす場合は Firebase の管理が必要なので、システム担当者に依頼してください（オーナー判定は `owner` のみ。それ以外のIDはすべてスタッフ扱いになります）。

## 15. 運用上の注意（重要）

1. **開いている伝票は自動保存されます。** 未会計の注文も保存されるため、リロードしたり別端末で同じURLを開いても、同じ卓・同じ注文が表示されます。
   - 2台で**同じ卓を同時に**編集した場合は、後から操作した内容が優先されます。
2. **ネット接続が必要です。** Wi-Fi が不安定だと保存や同期に失敗することがあります。
3. **会計・取引の削除は元に戻せません。** 削除・編集は慎重に。
4. オーナーのパスワードは厳重に管理してください（売上・バックが見られます）。
5. **アプリを更新したら、iPad は一度“開き直し”を**（タブを閉じて開く）。古い画面が残っていると、まれに表示が食い違うことがあります。

## 16. 困ったとき

| 症状 | 対処 |
|---|---|
| ログインできない | ID（`owner`/`staff`）とパスワードを確認。半角で入力。 |
| メニュー/キャストが空 | オーナーでログインし、各管理画面の「投入」ボタンを押す。 |
| 売上が出てこない | 期間（今日/今週/今月）と上部の日付、ネット接続を確認。 |
| 会計が「締め済み」で確定できない | その日はレジ締め済み。売上管理→レジ締め→「締め解除」。 |
| 取引を編集/削除できない | その日が締め済み。先に「締め解除」を。 |
| 日付がオレンジ・遡及入力になっている | トップの帯で「今日に戻す」（リロードでも今日に戻ります）。 |
| 打刻に不整合がある | 打刻管理→要修正サマリーで対象キャストを確認し、修正・追加。 |
| 卓が消えない/変な卓が出る | iPad を開き直す（タブを閉じて開く）。会計済みの卓は自動で消えます。 |

解決しない場合は、納品担当（システム担当者）にご連絡ください。
