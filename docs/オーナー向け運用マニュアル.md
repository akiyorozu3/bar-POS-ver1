# バーPOS オーナー向け 運用マニュアル

お店のオーナー様向けの運用説明書です。ログイン情報、日々の使い方、メニュー・キャスト・料率の設定、売上の見方をまとめています。

---

## 0. ログイン情報（最初に控えてください）

> ⚠️ この欄は納品時に実際の値を記入してください。パスワードは外部に漏れないよう管理してください。

| 項目 | 内容 |
|---|---|
| アプリのURL | `https://bar-pos-b1993.web.app` |
| オーナー用 ログインID | `owner` |
| オーナー用 パスワード | `password` |
| スタッフ用 ログインID | `staff` |
| スタッフ用 パスワード | `000000` |

> ⚠️ **セキュリティ注意**: 上記パスワードは推測されやすいため、できるだけ早めに変更を推奨します（特にオーナー）。
> 公開URLは誰でもアクセスできるため、オーナーのパスワードが破られると売上・バック額がすべて閲覧されます。
> 変更は Firebase Console → Authentication → 対象ユーザー → パスワード再設定 から行えます（変更後はこの表も更新してください）。

- iPad のホーム画面に追加すると、アプリのように使えます（Safari でURLを開く → 共有 → 「ホーム画面に追加」）。
- **オーナーとスタッフでアカウントが分かれています。** スタッフ用では売上やバックは見られません。
- オーナー用パスワードは、スタッフに教えないでください（売上が見られてしまいます）。

## 1. オーナーとスタッフの違い

| できること | スタッフ | オーナー |
|---|:--:|:--:|
| 注文入力 | ○ | ○ |
| 会計 | ○ | ○ |
| 売上管理（売上・客数・支払方法別） | × | ○ |
| キャストバックの確認 | × | ○ |
| メニューの追加・編集・削除 | × | ○ |
| キャストの追加・編集・削除 | × | ○ |

スタッフがログインすると、画面上部のメニューに「注文入力」「会計」だけが表示されます。

## 2. 画面の構成

ログイン後、画面上部のタブで切り替えます（オーナーは5つ、スタッフは2つ）。

- **注文入力** … 席ごとに注文を取る画面
- **会計** … 支払いを確定する画面。いつでも開けて、上部の「未会計の卓」チップで卓を切り替えられます
- **売上管理**（オーナー）… 売上・客単価・支払方法別・キャストバックの集計、CSV出力、手数料/バック率/消費税の設定、レジ締め
- **メニュー管理**（オーナー）… ドリンク等メニューの管理
- **キャスト管理**（オーナー）… 出勤キャストの管理

画面いちばん上の黒い帯に **「日付」**（記録される営業日）と「ログアウト」があります。日付は通常は今日。過去日に変えると帯がオレンジになり「遡及入力」モードになります（「今日に戻す」で戻せます／リロードでも今日に戻ります）。

## 3. 初期設定（導入時に一度だけ）

オーナーでログインして行います。

### 3-1. メニューを登録する
1. 上部「**メニュー管理**」を開く
2. まだ何も無ければ「**デフォルトメニューを投入**」を押すと、ウイスキー/カクテル/ビール/フード等の初期メニューが入ります
3. あとは画面から自由に追加・編集・削除（下記4参照）

### 3-2. キャストを登録する
1. 上部「**キャスト管理**」を開く
2. 「**初期キャストを投入**」を押すか、上の入力欄に名前を入れて「＋追加」

### 3-3. 消費税・手数料・バック率を設定する
1. 「**売上管理**」→「**手数料/バック**」ボタン
2. 以下をまとめて設定して「保存」
   - **価格・消費税の扱い**:「税込で登録・加算しない」/「税抜で登録・会計時に加算」
   - **消費税率**（%）
   - **カード手数料 / QR手数料**（%）
   - **卓バック率**（%）… 会計の合計にかけて卓の担当に入る
   - **キャストドリンクバック率**（%）… キャストドリンクの料金にかけてその担当に入る

> 詳しくは下の「6. 売上の見方」内の **消費税の設定** と **バックの考え方** を参照。

---

## 4. メニューの管理（オーナー）

「メニュー管理」画面で行います。

- **追加**: 上部でカテゴリを選び、メニュー名・価格を入れて「＋追加」。「本日限定」を選ぶと当日限りのメニューになります。
- **編集**: 一覧の各行で名前・価格を直接書き換え、「保存」を押す。
- **削除**: 各行の「削除」。
- カテゴリ: **セット / ショット / シャンパン / キャストドリンク / ウイスキー / カクテル / ビール / フード**。
- 価格の入力単位は消費税の設定で変わります（「税込で登録」モードなら**税込**、「税抜で登録」モードなら**税抜**。入力欄の表示も自動で切り替わります）。

> 本日限定メニューは、注文画面の「本日メニュー編集」からも追加・削除できます。
> 「キャストドリンク」は専用カテゴリです。このカテゴリの商品にだけ「ドリンクバック」が付きます（「6. 売上の見方」内のバックの考え方を参照）。

---

## 5. キャストの管理（オーナー）

「キャスト管理」画面で行います。

- **追加**: 名前を入れて「＋追加」（Enter でも追加可）。
- **名前変更**: 各行の名前を書き換えて「保存」。
- **削除**: 各行の「削除」。

注意:
- ここで登録したキャストが、注文画面の「卓の担当」やキャストドリンクの「ドリンク担当」の選択肢になります。
- **キャストを削除・改名しても、過去の売上やバック集計は変わりません**（記録時点の名前がそのまま残ります）。改名は、以降の新しい注文から反映されます。

---

## 6. 売上の見方（オーナー）

「売上管理」画面で確認します。

- 上部で **今日 / 今週 / 今月** を切替（「今週」は直近7日間）。
- **売上合計**（税込）/ 件数 / 客単価
- **決済手数料**（カード・QRの手数料合計）
- **実際の入金合計**（売上 − 手数料。実際に手元に入る金額）
- **支払い方法別 実入金**（現金・カード・QRごとの件数・売上・手数料・実入金）
- **支払い方法の割合**
- **キャストバック集計**（キャストごとの卓数・売上・バック額）

### 消費税の設定
「手数料/バック」パネルで、消費税の扱いと税率を設定できます。
- **税込で登録・加算しない**: メニューに登録した金額がそのままお客様の支払額。消費税は上乗せ・表示しません（登録額＝最終価格）。
- **税抜で登録・会計時に加算**: 登録は税抜。会計時に消費税率（%）を上乗せして合計を出します。
- **消費税率**（%）も設定可（既定10%、0%も可）。
- 切替に連動して、メニュー登録欄やレシートの「税込/税抜」表記も自動で変わります。

### バックの考え方（2種類）
バックは次の2つが**別々に**計算され、合算されます。

1. **卓バック**：会計の**合計 ×「卓バック率」** → その卓の担当キャストに入る。
   - 卓の担当が**複数いる場合は頭割り**（例: 合計3,200×10%=320を2名で 160ずつ）。
2. **キャストドリンクバック**：キャストドリンクの**料金 ×「ドリンクバック率」** → その**ドリンクの担当**に入る（卓バックとは別に上乗せ）。

ポイント:
- キャストドリンクの料金は**卓バックの合計にも含まれます**。つまり、たまの卓にりなのキャストドリンクがあれば、**たまには卓バック（ドリンク料金込み）、りなにはドリンクバック**が両方付きます。
- 卓の担当が未設定の卓は卓バックが付きません（集計には表示）。
- 例（卓バック10%・卓担当=たま）: セット2,000＋キャストドリンク（りな）1,200＝合計3,200 → **たま 320**／**りな = 1,200×ドリンク率**。

## 7. CSV 出力（オーナー）

「売上管理」画面の右上から、選択中の期間のデータを出力できます。

- **売上CSV** … 取引明細（日時・席・支払方法・売上・手数料・実入金・主担当など）
- **バックCSV** … キャスト別の担当件数・売上(税抜)・バック額

Excel やスプレッドシートでそのまま開けます（文字化けしません）。給与計算や帳簿に利用してください。

---

## 8. レジ締め（オーナー）

営業終了時に、その日の売上を確定します。「**売上管理**」→「**レジ締め**」ボタンで開きます。

- 本日の **現金 / カード / QR 売上・売上合計・決済手数料・実際の入金合計・バック合計・件数** が表示されます。
- **未会計の卓がある場合は警告**が出ます。先に会計を済ませてから締めてください。
- **「本日を締める」** を押すと確定し、**本日の会計入力ができなくなります**（スタッフが会計しようとしても「締め済み」でブロック）。
- 間違えた場合は **「締め解除」** で元に戻せます（オーナーのみ）。
- 締めた時点の金額は記録として保存されます。

> 「その日」は日付（営業日）ごとです。遡及入力で締め済みの日に入れようとしてもブロックされます。

---

## 9. スタッフを増やしたいとき

スタッフ用アカウントは「`staff`」1つを共有する想定です。別アカウントを増やす場合は Firebase の管理が必要なので、システム担当者に依頼してください（オーナー判定は `owner` のみ。それ以外のIDはすべてスタッフ扱いになります）。

---

## 10. 運用上の注意（重要）

1. **開いている伝票は自動保存されます。** 未会計の注文も保存されるため、リロードしたり、別の端末（PCや別のiPad）で同じURLを開いても、同じ卓・同じ注文が表示されます。
   - ただし2台で**同じ卓を同時に**編集した場合は、後から操作した内容が優先されます。
2. **ネット接続が必要です。** Wi-Fi が不安定だと保存や同期に失敗することがあります。接続を確認してください（完全オフラインでのリロードは未保証）。
3. **打ち間違いの会計を取り消す機能はありません。** 間違えて確定した場合は、システム担当者に依頼して修正してもらってください。
4. オーナーのパスワードは厳重に管理してください（売上・バックが見られます）。

## 11. 困ったとき

| 症状 | 対処 |
|---|---|
| ログインできない | ID（`owner`/`staff`）とパスワードを確認。半角で入力。 |
| メニュー/キャストが空 | オーナーでログインし、各管理画面の「投入」ボタンを押す。 |
| 売上が出てこない | 期間（今日/今週/今月）を確認。ネット接続を確認。 |
| スタッフなのに売上が見える/見えない | ログインIDを確認（`owner` はオーナー、それ以外はスタッフ）。 |
| 伝票が見当たらない | 別の卓を選んでいないか確認（注文は卓ごとに保存）。ネット接続も確認。 |
| 会計が「締め済み」で確定できない | その日はレジ締め済み。オーナーが売上管理→レジ締め→「締め解除」で入力可能に。 |
| 日付がオレンジ・遡及入力になっている | トップの帯で「今日に戻す」を押す（リロードでも今日に戻ります）。 |

解決しない場合は、納品担当（システム担当者）にご連絡ください。
